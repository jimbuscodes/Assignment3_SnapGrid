JavaScript Interactivity Assignment 3 20 Points

SnapGrid — Social Media Profile Dashboard

You're a front-end developer at SnapGrid, a new social media startup. The designers have handed you a fully built profile page — but nothing works yet. Your job is to wire up all the interactivity using JavaScript DOM manipulation and event handling.

📄 index.html provided — don't modify 🎨 styles.css provided — don't modify ✏️ solutions.js write your code here
Task 1

Dark / Light Mode Toggle

3 pts

Wire up the Dark / Light Mode button in the top navigation bar so it switches the whole app's theme each time it is clicked.

  • Select the button #themeToggleBtn, its icon #themeIcon, and its label #themeLabel
  • Declare a boolean: let isDark = true
  • Add a click event listener to the button
  • Inside the listener, flip isDark with !isDark, then:
    • Use document.documentElement.setAttribute('data-theme', ...) — set to "light" or "dark"
    • Light mode: set #themeIcon text to "☀️" and #themeLabel text to "Light Mode"
    • Dark mode: set them back to "🌙" and "Dark Mode"
💡 Hint: document.documentElement refers to the <html> element — the data-theme attribute on it controls all CSS variables for both themes.
Task 2

Follow / Unfollow with Live Counter

4 pts

Make the Follow button in the profile card toggle between following and not following — and update the follower count live.

  • Select #followBtn, #followBtnText, and #followerCount
  • Declare: let isFollowing = false and let followers = 1284
  • Add a click listener to #followBtn
  • Flip isFollowing each click, then:
    • Now following: increment followers by 1, set button text to "✅ Following", add class following to #followBtn
    • Unfollowed: decrement followers by 1, set text to "➕ Follow", remove class following
    • In both cases, update #followerCount.textContent to the new followers value formatted with toLocaleString()
💡 Hint: toLocaleString() on a number automatically adds comma separators — so (1285).toLocaleString() gives "1,285". Try it in the console!
Task 3

Live Post Composer

5 pts

Bring the post composer (text box at the top of the feed) to life. It should count characters in real time and dynamically create a new post card when submitted.

  • Select #composerTextarea, #charCounter, #postBtn, #postFeed, and #postCount
  • Add an input event listener to #composerTextarea:
    • Calculate remaining characters: 280 - textarea.value.length
    • Update #charCounter text to "X characters remaining"
    • If remaining < 20, add class counter-warning to #charCounter; otherwise remove it
    • Enable #postBtn if the trimmed value is not empty; otherwise keep it disabled
  • Add a click listener to #postBtn:
    • Get the text from textarea.value.trim()
    • Create a new <article> element with class post-card new-post
    • Set its data-tags attribute to any hashtags found in the text (you can leave this as an empty string for now)
    • Set its innerHTML to the post card template shown below
    • Use postFeed.prepend(newPost) to add it at the top of the feed
    • Reset textarea.value to "", reset #charCounter to "280 characters remaining", remove counter-warning, disable #postBtn
    • Increment and update #postCount

📋 New Post Card Template (use inside innerHTML):

<div class="post-header">
  <div class="post-avatar">🧑‍💻</div>
  <div class="post-meta">
    <span class="post-author">Alex Rivera</span>
    <span class="post-time">Just now</span>
  </div>
</div>
<p class="post-body">${postText}</p>
<div class="post-actions">
  <button class="like-btn" data-liked="false">
    <span class="like-icon">🤍</span>
    <span class="like-count">0</span>
  </button>
</div>
💡 Hint: Use a template literal (backtick string) for the innerHTML. After creating the new post, call attachLikeListeners() (from Task 4) so the new post's heart button works too!
Task 4

Like / Unlike Posts

4 pts

Make the heart button on every post card toggle between liked and not liked, update the like count, and keep a running total in the sidebar.

  • Write a named function attachLikeListeners() (Task 3 needs to call it after adding new posts)
  • Inside attachLikeListeners(), select all .like-btn elements and loop through them with forEach
  • For each button, replace any old listener by cloning: const fresh = btn.cloneNode(true); btn.parentNode.replaceChild(fresh, btn);
  • Add a click listener to the cloned fresh button:
    • Read the current liked state: fresh.getAttribute('data-liked') === 'true'
    • Toggle: if liked → set data-liked="false", subtract 1 from count, set icon to "🤍", remove class liked
    • If not liked → set data-liked="true", add 1 to count, set icon to "❤️", add class liked
    • Update .like-count span inside the button with the new number
    • After toggling, call updateTotalLikes()
  • Write a second function updateTotalLikes():
    • Count all .like-btn[data-liked="true"] elements
    • Update #totalLikedCount with that number
  • Call attachLikeListeners() once at the bottom of your file to wire up the initial posts
💡 Hint: The clone trick (cloneNode + replaceChild) removes all old event listeners from a button before adding a new one — this prevents a button from firing the same action multiple times if attachLikeListeners() is called more than once.
Task 5

Hashtag Filter

4 pts

Make the tag pills in the sidebar filter the post feed — clicking a tag shows only posts that include that hashtag.

  • Select all .tag-pill elements and #filterResultMsg
  • Loop through the pills with forEach and add a click listener to each
  • Inside the listener:
    • Remove class active from every pill, then add it to the clicked pill only
    • Read the filter value: const tag = pill.getAttribute('data-tag')
    • Select all .post-card elements in #postFeed
    • Loop through posts:
      • If tag === "all", remove class hidden from every post
      • Otherwise, check if the post's data-tags attribute includes(tag):
        • If yes → remove class hidden
        • If no → add class hidden
    • Count visible posts: filter the posts array where !post.classList.contains('hidden')
    • Update #filterResultMsg:
      • If tag is "all": "Showing all X posts"
      • Otherwise: "X post(s) tagged ${tag}"
💡 Hint: querySelectorAll() returns a NodeList, not an array. Use Array.from(document.querySelectorAll('.post-card')) to convert it so you can use .filter() and .length on it easily.
